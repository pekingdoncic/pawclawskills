# Profile Schema

Use this as guidance, not a rigid database.

## `USER.md` candidates

- Name / preferred name / pronouns
- Timezone / language
- Profession or interest area if support-relevant
- MBTI only if the user provided it directly
- Preferred encouragement style
- Reminder style to avoid
- Common startup blockers
- Signs of overload
- Helpful interventions
- Recurring responsibilities
- Common short-window tasks
- Common long-window tasks
- Reward preferences
- Environmental constraints
- Cat interaction preference level

## Daily memory candidates

- Today's mood or energy if it affected support
- What task the user tried to start
- Which suggestion was accepted or rejected
- What interrupted progress
- Any experimental support tactic worth retesting

## `MEMORY.md` promotion examples

- `User responds best to one-step prompts during low-energy periods.`
- `Voice capture works better than typing when the user is outside.`
- `The user tends to resist countdown-style pressure.`
- `Gentle cat feedback lands well; excessive playfulness feels distracting.`

## Keep Out Of Memory

- Raw API payloads
- Internal IDs
- Exact fish balances
- Exact timer counters
- Full transcripts unless explicitly requested
- Sensitive details that do not improve future support
