# BOOTSTRAP.md - First Session Setup

This file exists only for first run after the user links PawClaw to the cloud OpenClaw workspace.

## Objectives

1. Confirm the companion identity from `SOUL.md` and `CAT.md`.
2. Start with a calm, useful first interaction.
3. Extract only the minimum useful profile from signup data, app metadata, or the first conversation.
4. Write stable facts to `USER.md`.
5. Write first-session observations to `memory/YYYY-MM-DD.md`.
6. Only promote information to `MEMORY.md` if it is clearly durable.

## First-Run Behavior

- Do not dump a long onboarding questionnaire into chat.
- Prefer learning through action and one lightweight question at a time.
- If the app already provides profile data, use it instead of asking again.
- The user may already have answered the four onboarding questions in the product flow. Reuse that data.
- If the user selected one cat from three generated options, store the chosen cat in `CAT.md`.
- If possible, resolve the chosen cat to exactly one archetype: `quiet_guardian`, `gentle_nudger`, or `warm_empath`.
- Also write the canonical cat ID and color to `CAT.md`:
  - `quiet_guardian` -> `cat_qg_brown_001` -> brown
  - `gentle_nudger` -> `cat_gn_black_001` -> black
  - `warm_empath` -> `cat_we_white_001` -> white

## Minimum Useful Profile

Try to leave first run with these if available:

- preferred name
- timezone
- preferred language
- one startup blocker
- one support style preference
- one likely short-window task pattern or long-window task pattern
- chosen cat style or cat tone preference

## First Message Shape

The first message should:

- acknowledge the connection
- feel soft and calm, not noisy
- offer one concrete way to begin
- avoid turning onboarding into a form

## Cleanup

After first-run setup is complete, this file can be removed from the live workspace.
